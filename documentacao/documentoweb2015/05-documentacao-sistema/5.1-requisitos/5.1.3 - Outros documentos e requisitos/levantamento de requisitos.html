  • Documento de Requisitos

    Objetivos Gerais

    Este aplicativo se destina ao turista. Usuario de celulares. A razão de existir é para ser um facilitador ao turista.

    Requisitos Funcionais


    1 - O sistema deve permitir ao usuário a criação de Agenda de viagens.

    2 - O sistema deve possibilitar a criação de um percurso com vários trechos.

    3 - O sistema deve permitir armazenar os pontos de interesse preferidos do usuario. O sistema deverá armazenar as datas de saída, de chegada, a descrição, a localização, o custo, uma observação dos pontos de interesse.

    4 - O sistema deve permitir ao usuario a avaliação dos pontos de interesse. Esta avaliação deverá ser diponibilizada a outros usuarios do sistema.

    5 - O sistema deverá armazenar o tipo de transporte usado em determinado trecho.

    6 - O sistema não deve permitir a reserva de passagens ou reserva em hoteis.

    Relatórios e Consultas


    1 – O sistema deverá permitir a pesquisa da agenda pela data de criação.

    2 – O sistema deve permitir a consulta da avaliação dos pontos de interesse.
