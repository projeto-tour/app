ESPECIFICAÇÃO DE CASO DE USO-UC01

UC01 – EFETUAR LOGIN

ATORES DESCRIÇÃO
Usuário Usuário com permissão de consulta
Administrador do Sistema Usuário com permissão completa no sistema

Objetivo

Permitir ao usuário cadastrar-se para adicionar e gerenciar suas agendas de viagens e iniciar o processo de localização de pontos de interesse.

Ator Primário: CLiente

Pré Condições

Não se aplica

Pós Condições

Não se aplica

Fluxo Principal

FP - Efetuar Login
PASSO INTERAÇÃO DO ATOR
1 Sistema apresenta a tela de login
2 Usuário digita e-mail
3 Usuário digita senha
4 Usuário seleciona botão Login
5 Sistema valida informações
6 Sistema exibe tela Pontos de Interesse
7 O caso de uso é encerrado

Fluxo Alternativo

FA01 - Novo Usuário
PASSO INTERAÇÃO DO ATOR
1 Sistema apresenta a tela de login
2 Usuário digita e-mail
3 Usuário digita senha
4 Usuário seleciona botão Criar
5 Sistema verifica informações
6 Sistema redireciona usuário para tela Pontos de Interesse
7 O fluxo alternativo é encerrado


FA02 - Usuário escolhe conectar-se utilizando conta do Google+
PASSO INTERAÇÃO DO ATOR
1 Sistema apresenta a tela de login
2 Usuário decide conectar-se utilizando conta do Google+
3 Sistema exibe tela de Login da conta do Google+
4 Usuário informa e-mail e senha da conta do Google+
5 Sistema solicita autorização para utilizar as informações do Google+
6 Sistema recupera informações para que a conta seja habilitada
7 Sistema redireciona usuário para tela Pontos de Interesse
8 O fluxo alternativo é encerrado


FA03 - Usuário escolhe Pular Cadastro
PASSO INTERAÇÃO DO ATOR
1 Sistema apresenta a tela de login
2 Usuário seleciona botão Pular
3 Sistema exibe tela Pontos de Interesse
4 O fluxo alternativo é encerrado

ESPECIFICAÇÃO DE CASO DE USO-UC02

UC02 – CADASTRAR PONTOS DE INTERESSE

ATORES DESCRIÇÃO
Usuário Usuário com permissão de consulta
Administrador do Sistema Usuário com permissão completa no sistema

Objetivo

Permitir ao usuário cadastrar Pontos de Interesse em diversas localidades.

Ator Primário: CLiente

Pré Condições

O usuário deverá estar cadastrado e logado no sistema.

Pós Condições

Não se aplica

Fluxo Principal

FP - Seleção de Pontos de Interesse
PASSO INTERAÇÃO DO ATOR
1 Sistema apresenta tela de Pontos de Interesse
2 Usuário seleciona o Tipo de Ponto de Interesse
3 Usuário digita palavra-chave do Ponto de Interesse
4 Sistema disponibiliza o Ponto de Interesse escolhido no Mapa
5 Usuário seleciona Ponto disponibilizado no Mapa
6 Sistema disponibiliza tela com descrição do Ponto de Interesse
7 Usuário seleciona Botão Salvar
8 Sistema informa que as informações salvas com sucesso
9 O caso de uso é encerrado

Fluxo Alternativo

FA01 - Novo Ponto de Interesse
PASSO INTERAÇÃO DO ATOR
1 Passos de 1 a 8 FN é refeito
2 Sistema disponibiliza a tela com a lista de Pontos de Interesse executado no item 1 do FA01
3 O fluxo alternativo e encerrado

ESPECIFICAÇÃO DE CASO DE USO-UC03

UC03 – CRIAR AGENDA

ATORES DESCRIÇÃO
Usuário Usuário com permissão de consulta
Administrador do Sistema Usuário com permissão completa no sistema

Objetivo

Permitir ao usuário cadastrar agendas de viagens organizando por ponto de interesse.

Ator Primário: CLiente

Pré Condições

O usuário deverá estar cadastrado e logado no sistema.

Pós Condições

Não se aplica

Fluxo Principal

FP - Pesquisar Agenda
PASSO INTERAÇÃO DO ATOR
1 Sistema apresenta tela de Pontos de Interesse
2 Sistema apresenta tela com a lista de Agendas cadastradas
3 Usuário seleciona Pesquisa
4 Usuário digita palavra-chave
5 Sistema valida informações
6 Sistema exibe Agenda pesquisada
7 O caso de uso é encerrado

Fluxo Alternativo

FA01 - Nova Agenda
PASSO INTERAÇÃO DO ATOR
1 Passos 1 e 2 iguais ao FP
2 Usuário seleciona botão de Criar de Agendas
3 Sistema disponibiliza tela de Criar Agendas
4 Usuário seleciona botão Tipo de agenda
5 Sistema disponibiliza tela com a lista de Tipo de Agenda
6 Usuário seleciona o Tipo de Agenda e confirma
7 Usuário digita descrição da Agenda
8 Usuário insere data inicial e data final da Agenda
9 Usuario confirma a inclusão da Agenda
10 Sistema direciona usuário para tela de Rotas
11 O fluxo alternativo é encerrado


FA02 - Menu de Opções - Excluir
PASSO INTERAÇÃO DO ATOR
1 Sistema apresenta de Agendas cadastradas
2 Usuário seleciona Agenda
3 Usuário seleciona botão de Menu
4 Sistema sistema disponibiliza opção Excluir, Editar e Cancelar
5 Usuário seleciona a opção Excluir
6 Sistema informa usuário da correta exclusão da Agenda
7 O fluxo alternativo e encerrado


FA03 - Menu de Opções - Editar
PASSO INTERAÇÃO DO ATOR
1 Passos de 1 a 4 iguais ao FA02
2 Usuário seleciona a opção Editar
3 Sistema redireciona usuário para Tela Editar Agendas
4 Usuário altera informações desejadas e confirma
5 Sistema informa usuário da correta alteração da Agenda
6 Sistema exibe tela Criar Rotas
7 O fluxo alternativo é encerrado

UC04 – CRIAR ROTA

ATORES DESCRIÇÃO
Usuário Usuário com permissão de consulta
Administrador do Sistema Usuário com permissão completa no sistema

Objetivo

Permitir ao usuário criar rotas de viagens.

Ator Primário: CLiente

Pré Condições

O usuário deverá estar cadastrado e logado no sistema

Pós Condições

Não se aplica

Fluxo Principal

FP - Nova Rota
PASSO INTERAÇÃO DO ATOR
1 Sistema apresenta de Rotas
2 Usuário seleciona botão de cadastro de Rotas
3 Sistema disponibiliza tela de Criar rotas
4 Usuário digita o ponto de saída
5 Usuário seleciona a data de saída desejada
6 Usuário digita o ponto de chegada
7 Usuário seleciona a data de chegada desejada
8 Usuário seleciona o ponto de interesse
9 Usuário seleciona o tipo de meio de transporte
10 Usuário Salva informações
11 USistema informa que as informações foram gravadas com êxito
12 O fluxo alternativo e encerrado


FA02 - Exibir Rotas
PASSO INTERAÇÃO DO ATOR
1 Passo de 1 a 6 do FP do caso de uso UC03 – Criar Agenda
2 UUsuário seleciona botão Rotas da Agenda
3 Sistema exibe tela com as informações sobre a Rota
4 O fluxo alternativo é encerrado


FA03 - Editar Rotas
PASSO INTERAÇÃO DO ATOR
1 SSistema apresenta de Rotas
2 Usuário seleciona a opção Editar
3 Sistema redireciona usuário para Tela Editar Rotas
4 Usuário altera informações desejadas e salva
5 Sistema informa usuário da correta alteração na Rota
6 Sistema exibe tela inicial de Rotas
7 O fluxo alternativo e encerrado


FA04 - Excluir
PASSO INTERAÇÃO DO ATOR
1 Sistema apresenta de Rotas
2 Usuário seleciona a opção Excluir
3 Sistema informa Exclusão com sucesso
4 O fluxo alternativo é encerrado
