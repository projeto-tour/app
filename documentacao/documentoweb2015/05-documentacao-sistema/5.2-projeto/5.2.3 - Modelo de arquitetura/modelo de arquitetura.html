Documentação de Software do projeto Partiu!

  • Modelo de Arquitetura

    Abaixo é descrita a arquitetura do projeto, baseada em suas tecnologias e comunicação entre componentes e nuvem do Firebase.

    A aplicação foi desenvolvida de forma híbrida, como um projeto Web, respeitando as diretivas de uma aplicação baseada em Progressive Web Apps. Dessa forma, temos a interface definida para o aplicativo, baseado no framework Ionic 2, tendo como ferramenta de backend, responsável pelas regras de negócio, o framework Angular2, o mais novo framework da Google. Ainda no escopo do dispositivo, temos o AngularFire, que encapsula a Firebase SDK e nos permite fazer com o que a aplicação esteja diretamente conectada, na nuvem, ao Firebase, que é a ferramenta responsável por todo o gerenciamento do aplicativo.

    Alinhadas as funcionalidades nativas do dispositivo móvel, temos o Web App Manifest, o Service Workers e App Shell, para que seja possível que a aplicação utilize a especificação da Progressive Web Apps.

    Todo o gerenciamento de dados do aplicativo é baseado nas tecnologias Google, que provê ferramentas maduras para esse fim, além de proporcionar um alto grau de estabilidade e permissionamento para garantirmos alta disponibilidade dos serviços.

    O aplicativo Partiu! utiliza as mais recentes tecnologias para prover uma melhor experiência aos nossos usuários.
